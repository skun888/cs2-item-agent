# CSQAQ 个人 Token 权限审计

CSQAQ 在项目中定位为可选增强数据源，不替代 SteamDT，也不替代项目自己的公开 Steam 库存快照。

## 配置

在被 Git 忽略的本地 `.env` 中填写：

```dotenv
CSQAQ_API_TOKEN=你的个人Token
CSQAQ_BASE_URL=https://api.csqaq.com
```

使用前需要在 CSQAQ 绑定当前公网 IP。不要把 Token 发到聊天、截图、Issue 或 Git 仓库。

执行：

```bash
npm run dev -- provider audit csqaq
```

审计按照不快于 1.1 秒一次请求的节奏串行执行。它只输出 HTTP 状态、提供方状态码、数量和字段名，不输出或保存 Token、SteamID、昵称、头像及持仓内容。

## 状态含义

| 状态 | 含义 |
| --- | --- |
| `available` | 当前个人 Token 本次调用成功 |
| `authentication_failed` | Token 无效、过期或认证失败 |
| `configuration_required` | 通常表示尚未绑定当前白名单 IP |
| `permission_denied` | 接口存在，但个人 Token 没有权限 |
| `contract_rejected` | 请求参数或文档契约发生变化，不能等同于无权限 |
| `rate_limited` | 请求被限流 |
| `provider_rejected` | 提供方拒绝，但原因未能可靠归类 |
| `unavailable` | 提供方服务异常 |
| `network_error` | 请求未取得响应 |
| `not_probed` | 出于企业权限、超大响应或暂停更新等原因主动跳过 |

## 当前审计范围

- 个人批量出售价格；
- 单件饰品近 180 天存世量走势；
- 挂刀候选；
- 板块指数、指数 K 线和每日 Steam 美金卡价；
- 收藏品目录及单收藏品包含物；
- 武器箱开箱量与 ROI；
- 库存监控任务和最新动态；
- 持有量排行榜的个人 Token 实际权限；
- 明确标为企业的大体量接口只登记为 `not_probed`，不会下载；
- 文档标为暂停更新的实时成交接口不调用。

`available` 只证明审计当时可访问，不保证未来额度、字段或条款不变。CSQAQ 的监控数据不能描述为 Steam 全网数据。

## 2026-07-20 个人 Token 真实结果

本次只读审计共发送 8 个请求，全部返回 HTTP 200 与提供方状态码 200：

- 批量出售价格可用；
- 单件饰品近 180 天存世量返回 180 个数据点；
- 挂刀候选返回 50 条；
- 武器箱开箱统计返回 43 条；
- 武器箱与胶囊 ROI 返回 478 条；
- 库存监控任务列表可用；
- 库存监控最新动态可用；
- 单件饰品持有量排行榜可用，测试饰品返回 143 条。

因此，持有量排行榜虽然权限归类此前存在疑问，但已确认当前个人 Token 能实际调用。项目仍需表述为“CSQAQ 已监控范围内的排行”，并保留未来权限策略变化的降级处理。企业全量饰品 ID/价格未请求，暂停更新的实时成交接口也未请求。

同日使用 `Danger Zone Case` 完成双源兼容验收：SteamDT 返回 9 个平台，CSQAQ 返回 BUFF、悠悠有品和 Steam 3 个平台。BUFF 价格与在售量一致；悠悠价格一致、在售量相差 3；Steam 价格相差约 4.01%，且 SteamDT 的 Steam 数据时间比请求时间早约 54 分钟。该结果证明双源数据必须保留各自来源和时间，不能静默覆盖。

## 与 SteamDT 的兼容规则

两个来源统一为以下关键字段：

```text
marketHashName + platform + provider + source + observedAt
```

同一平台的 SteamDT 与 CSQAQ 报价并列保留，不相互覆盖。价格差异需要连同各自刷新时间和口径展示，不能自动解释为可成交套利。CSQAQ 挂刀候选只作为候选来源，最终手续费和七日情景必须由项目的可修改费率模板重新计算。

## 2026-07-22 增量真实验收

- 首页数据成功返回板块列表和当日 Steam 卡价 `¥503.6/100 USD`；
- 千战指数日 K 线成功返回自 2023 年起的 OHLC，项目计算出 1/7/15/30 日表现；响应中的成交量字段为 0，因此只作为指数走势使用；
- 收藏品列表返回 547 项，能够区分普通收藏品与赛事纪念包；纪念品按 Valve 2026-05-20 后的规则标为可输入、移除纪念属性，并在缺少基础收藏品映射时保持输出篮子未知；
- 普通“手套武器箱”详情在一次临时非 JSON 后重试成功，同步 202 项并验证喧嚣杀戮/炎龙之焰与稀有特殊物品层级；“核子危机收藏品”的失败仍按提供方响应记录，不生成伪关系；
- 两种挂刀方向均真实调用成功：“获得 Steam 余额”不使用卡价，“获得平台余额”读取当日卡价并分别应用 Steam 买入与国内平台退出参数；
- 提供方偶发返回非 JSON，客户端会显式失败，重试成功前不会冒充为新鲜数据。
