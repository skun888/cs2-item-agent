# 安全策略

CS2 Item Agent 仍处于公开预发布阶段。安全修复优先进入 `main`，并在后续预发布版本中交付；预发布版本暂不承诺长期维护周期。

## 私密报告

密钥泄露、任意命令执行、路径穿越、依赖供应链、未授权外部通知、Steam 隐私绕过或个人库存泄露等问题，请使用 GitHub 仓库 **Security** 页面中的 **Report a vulnerability** 私密报告入口。不要把漏洞细节、Key、Token、Webhook、Cookie、带认证信息的代理地址、真实 SteamID 或库存明细提交到公开 Issue。

如果私密报告入口暂不可见，请只提交一条不含漏洞细节和秘密的普通 Issue，说明“需要私密安全联系方式”，等待维护者提供安全渠道。已经泄露的凭据应立即在对应服务商处撤销或轮换，不要等待项目回复。

报告建议包含受影响版本或 Commit、操作系统、Node.js 版本、最小复现步骤、影响范围和已脱敏证据。维护者会先验证问题，再决定修复、缓解、上游报告和版本发布方式。

## 安全边界

- 项目不需要 Steam 密码、Cookie 或 Publisher Key；
- API Key、Token、Webhook 和代理只应保存在本地 `.env`；
- 项目不执行购买、出售、挂单、接受交易或隐私绕过；
- 数据源错误、限流、私密或不完整响应必须保持为未知状态；
- 企业微信测试和其他外部通知只有在用户明确要求时才可发送。

普通功能缺陷、行情覆盖或提供方临时故障可通过公开 Issue 报告，但同样必须先移除秘密和个人数据。
